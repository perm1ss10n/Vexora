# Vexora — Архитектура платформы

## Общая схема

[Sensors]
    ↓
[Vexora Box]
    ↓ (Wi-Fi / GSM)
[MQTT Broker]
    ↓
[Vexora Cloud]
    ├─ REST / gRPC API
    ├─ Provisioning
    ├─ OTA
    ├─ Alerts
    ↓
[InfluxDB]
    ↓
[Vexora App / Grafana]

## Принципы архитектуры
- Backend является **единственным источником истины**
- MQTT используется **только как транспорт**
- Устройство должно быть **fail-safe**
- Все компоненты проектируются с учётом **multi-tenant**
- Потеря связи не должна приводить к потере данных или управляемости

## Поток данных
1. Устройство собирает телеметрию
2. Публикует данные через MQTT
3. Backend валидирует и маршрутизирует данные
4. Телеметрия сохраняется в InfluxDB
5. Приложение и Grafana получают данные через API
